# Contributing to the DALIA Scale
Thank you for your interest in improving DALIA! Because this is a standard used to evaluate creative work, changes must be made methodically.

## How to Contribute
For minor typos or formatting issues, feel free to open a standard Pull Request (PR).

For substantial changes (e.g., proposing a new evaluation dimension, altering the definition of a level, or adding a discipline-specific matrix), you must use the RFC (Request for Comments) process.

## The RFC Process
1. **Fork the repository:** Create a copy of this repository on your GitHub account.
2. **Draft your proposal:** Copy the 0000-rfc-template.md (located in the /rfcs folder) and name it something descriptive like 0001-audio-generation-criteria.md.
3. **Submit a Pull Request (PR):** Open a PR with your draft. This signals that the RFC is in the "Pending Comment" stage.
4. **Discuss and Refine:** The community will review your RFC and leave comments. Be prepared to revise your proposal as folks weigh in.
5. **Final Comment Period (FCP):** Once the discussion settles and consensus is likely, a maintainer will apply the "Final Comment Period" label.
6. **Acceptance:** If no major objections remain after the FCP, the RFC will be merged into the active specification.
